# Thema 4 - Lab-Übung 1

Tag: Tag 3

Thema: Migration zur AWS Cloud

Übung: Rehost-Migration auf EC2 erklären

Verwendetes Go-Deploy-Lab: Lab 03

Nutze die Linux EC2 Instance aus Lab 3 als Beispiel für eine einfache Rehost-Migration. Stell dir vor, eine kleine interne Anwendung der Logistik Huber GmbH läuft bisher auf einem alten Linux-Server im Keller und soll zunächst möglichst unverändert in AWS laufen. Notiere, welche EC2-Bausteine du für dieses Ziel brauchst: AMI, Instance Type, Key Pair, Security Group und Zugriff per SSH.

Schreibe danach eine kurze Rehost-Checkliste mit mindestens sechs Punkten. Dazu gehören Betriebssystem, benötigte Ports, Login-Verfahren, öffentliche oder private Erreichbarkeit, Datenablage und Zuständigkeit für Updates. Ergänze am Ende zwei Grenzen des Rehost-Ansatzes: Was ist nach dem Umzug noch nicht automatisch modernisiert oder optimiert? Falls du schon Admin-Erfahrung hast, ergänze eine typische Rückfrage aus der Praxis. Wenn du neu bist, formuliere stattdessen eine Frage, die du dem Kunden vor der Migration stellen würdest.
